帮朋友做的简单的餐厅系统，下面是简单的需求，系统正在开发中，欢迎大家fork。

1.店家建立流程
首先系统后面页面为店家创建用户名和初始密码，店家登录后，输入店家介绍（地址， 特点，停车位，资历等），系统后台为此店家创建唯一ID并和用户ID绑定，以后此用户ID进入系统即进入店家资料或菜品建立（见流程2）相应页面，不能增删改查其他店家的资料。

涉及页面：店家用户管理页面（店家），店家资料建立页面（店家）。
涉及数据：店家用户ID和资料，店家ID和资料， 用户ID与店家ID绑定

2. 店用户建立
用户ID, 姓名，密码，岗位

3：菜品建立流程
店家员工登录后（不登陆无法识别是哪家店），系统自动识别是哪个店家，进入相应店家菜品输入页面，包括名称，介绍，材料, 口味（辣酸甜苦等级），大致完成时间,照片等，照片为高清照片（为省流量，应限制店家上传或系统自动缩小图片容量），页面应自动以缩略图显示，点击缩略图后看大图。

涉及页面：菜品建立页面（店家）
涉及数据：店家ID，店家用户ID， 菜品ID。 店家ID与店家用户ID 1:N, 店家ID与菜品ID 1：N. 

每个表都需要创建时间，创建人，修改时间，修改人。
